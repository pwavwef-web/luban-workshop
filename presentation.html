Digital Menu System

Customer Feature

How the Menu Works

  • Menu items are stored in Firebase Firestore database.
  • Anyone can browse the menu without logging in.
  • Each dish shows its name, photo, price, category, and description.
  • Dishes can be toggled available/unavailable by admins in real-time — changes appear instantly for customers.
  • A QR code is provided for quick scanning to the menu page.

Menu Categories

Soups Starters Beef & Lamb Pork Chicken Seafood Rice Noodles Dumplings Vegetable Mains

Menu Features

  • Category filters – click any category to show only those dishes.
  • "All" filter – shows every available dish at once.
  • Real-time visibility – hidden dishes disappear automatically from the live site the moment an admin toggles them off.
  • Dish photos – each item has a dedicated food photograph stored in the repository.
  • Standalone page (menu.html) – printable full menu layout, also linked via QR code on the physical menu card.

Admin Menu Control

  • Admins can hide/show any dish instantly from the Admin Dashboard → Menu Manager tab.
  • Useful for sold-out items or seasonal specials.
  • Changes are live immediately — no page refresh needed for customers.

Online Ordering System

Customer Feature

How to Place an Order

  1. Sign in – customers must be logged in to place an order.
  2. Browse the menu and click "Add to Cart" on any dish.
  3. Open the Cart Drawer (shopping bag icon in the navigation bar).
  4. Review items, adjust quantities, and view the subtotal.
  5. Click "Place Order" to submit.
  6. Order is saved to Firestore and appears in the Admin Dashboard immediately.

Order Management for Customers

  • "My Orders" section on the homepage (only visible when signed in).
  • View order history with status updates.
  • Orders can be cancelled within 5 minutes of placing; after that the kitchen has started preparing.
  • Each order shows: items ordered, quantity, total price, and current status.

Order Status Flow

Pending → Preparing → Ready / Served
  • Admins update order status from the Orders tab in the Admin Dashboard.
  • Customers see live status changes in My Orders.
  • Admins receive a notification badge on the Orders tab when a new order arrives.

Table Reservation System

Customer Feature

Making a Reservation

  • Available directly on the homepage (Reservation section) and via the navigation menu.
  • No login required to make a reservation — anyone can book a table.
  • Reservations are saved instantly to Firestore.

Reservation Form Fields

  • 📛 Full Name (required)
  • 📞 Phone Number (required)
  • 📧 Email Address (optional)
  • 📆 Date (required — up to 3 months ahead)
  • 🕐 Time (required)
  • 👥 Number of Guests (1 – 8+, required)
  • 💬 Special Requests — dietary needs, special occasions, etc.

Cancellations & Modifications

  • To cancel or change a booking, customers call 020 543 8455 or use the Contact Us form.
  • Cancellations must be made at least 24 hours before the booking time.

How Admins Manage Reservations

  • All reservations appear in the Reservations tab of the Admin Dashboard in real-time.
  • A pulsing red badge on the tab alerts admins to new bookings.
  • Admins can view, confirm, update, or delete any reservation.
  • Reservation data includes: guest name, phone, email, date, time, guest count, and special requests.
  • Only admins can read reservation data — customer privacy is protected by Firestore security rules.

User Authentication & Accounts

Security

Customer Accounts

  • Powered by Firebase Authentication.
  • Customers sign up / log in using an email address and password.
  • Account creation is free and requires only an email.
  • A "Sign Up / Login" button appears in the navigation bar for guests; it changes to show the user's email when logged in.
  • Logging out returns the user to the guest view.

What Logged-In Customers Can Do

  • ✅ Place orders online
  • ✅ View "My Orders" order history
  • ✅ Cancel orders within 5 minutes
  • ✅ Make table reservations

What Guests (Not Logged In) Can Do

  • ✅ Browse the full menu
  • ✅ Make a table reservation
  • ✅ View all public pages (FAQ, Contact, Staff Guide)
  • ❌ Cannot place orders
  • ❌ Cannot view "My Orders"

Auth Flow on the Homepage

  1. Guest clicks "Sign Up / Login".
  2. A modal dialog opens with fields for email and password.
  3. On success, the navbar updates to show the user's email and a logout button.
  4. If the user has admin privileges, an "Admin Dashboard" link appears automatically.

Admin Login & Access Control

Administration

How to Access the Admin Dashboard

  1. Navigate directly to admin.html.
  2. Enter your admin email address and password.
  3. Click "Sign In".
  4. The system verifies admin status — non-admin users are automatically signed out with an "Access Denied" message.
  5. On success, the full dashboard is revealed.

Three Ways to Be Recognised as Admin

  • 🔑 Firebase Custom Claims — admin: true set via the Firebase Admin SDK (recommended for production; most secure).
  • 📧 Bootstrap email — account with email admin@luban.com (for quick development/testing).
  • 📋 Firestore admins collection — email present in the /admins database collection (manageable via the Admin Users tab).

Granting Admin Access to New Users

  1. Log in to admin.html as an existing admin.
  2. Click the "Admin Users" tab in the sidebar.
  3. Enter the email address of the new admin.
  4. Click "Grant Admin Access" — the email is added to the Firestore admins collection.
  5. The new user can now log in to admin.html with full privileges.

Revoking Admin Access

  • In the Admin Users tab, a list of all current admins is displayed.
  • Click "Revoke" next to any email to remove that person's admin access immediately.
  • The revoked user will be denied access on their next login attempt.

Team Profiles, Contact & Technology

Backend & More

Team & Staff Directory

  • team/index.html — public team directory; shows all approved staff profiles.
  • team/profile-form.html — staff members submit personal profiles (name, role, bio, photo, socials).
  • team/profile.html — individual public profile page for each team member.
  • team/review.html — admin-only panel; review, approve, or reject submitted profiles before they go live.
  • Profiles are stored in the teamProfiles Firestore collection; only approved ones are visible to the public.

Contact Us Page

  • Visitors submit a message with name, email, subject, and message.
  • Messages are saved to the contact_messages Firestore collection.
  • Admins read and manage messages from the Messages tab in the dashboard.
  • No authentication required to send a message.

Technology Stack

  • 🔥 Firebase Authentication — user sign-up, login, and session management.
  • 🔥 Firebase Firestore — real-time database for menu, orders, reservations, messages, and team profiles.
  • 🎨 Tailwind CSS — responsive, utility-first styling.
  • ✨ Lucide Icons — clean SVG icon set used throughout the site.
  • 🌐 HTML5 / Vanilla JavaScript — no frameworks; lightweight and fast.
  • 📦 GitHub — version control and deployment via GitHub Pages / Azure.

Security Highlights

  • Firestore Security Rules enforce that only admins can read reservations, orders, and messages.
  • Admin dashboard verifies admin status server-side before showing any data.
  • Non-admin users are automatically signed out of admin.html.
  • Two rules files: development (firestore.rules) and production (firestore.rules.production) for different security levels.
  • Production uses Firebase Custom Claims — cannot be spoofed.
